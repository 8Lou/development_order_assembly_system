# Разработка системы сборки заказа.

## Задание

### Система должна включать в себя базу данных на MS SQL и клиентскую часть на Visual Studio C#. Для реализации потребуется:

- умение работать с таблицами реляционной базы,
- написание хранимых процедур и функций,
- составление форм клиентской части и привязку,
- добавление и обновление данных из базы.

---

Исходные данные:

1. Номенклатура бывает 2-х типов: запчасти и комплекты.
2. Комплекты состоят из нескольких запчастей. Запчасти неделимы.
3. Номенклатура собирается на нескольких сборочных площадках.
4. Каждая сборочная площадка собирает свой определенный вид номенклатуры. Это могут быть как запчасти, так и комплекты. Для сборки каждой номенклатуры определен срок сборки.
5. Для начала работ на сборку должен поступить заказ с номером и датой.
6. Заказ состоит из некоторого количества разной номенклатуры и может включать как запчасти, так и комплекты.
7. Заказы могут отменяться. В этом случае уже произведенная номенклатура хранится на сборочных площадках в качестве «свободных остатков».
8. Выполненные заказы также хранятся на сборочных площадках, но в «заказе».
9. Для сборки заказов сначала используются «свободные остатки», и только если их нет, - задание на сборку.
10. Разрабатываемая система должна быть способна отвечать на вопросы:

- успеют ли сборочные площадки собрать заказ,
- сколько номенклатуры в «свободных остатках» на каких площадках,
- сколько запланировано сборок для каждой площадки?

---

План

1. создание базы данных на MS SQL. Нужно создать таблицы для номенклатуры, комплектов, сборочных площадок, заказов, свободных остатков и других необходимых сущностей.

2. написанию хранимых процедур и функций, которые будут обеспечивать функционал системы, такие как добавление заказов, отмена заказов, расчет времени сборки и другие операции.

3. создание клиентской части на Visual Studio C#. Нам нужно создать формы для ввода заказов, отображения информации о свободных остатках на площадках, планирования сборок и т. д.

4. привязка клиентской части к базе данных, чтобы обеспечить добавление и обновление данных из базы при взаимодействии с пользователем.

Таким образом, создавать полнофункциональную систему сборки заказа, которая будет отвечать на поставленные вопросы и эффективно управлять процессом сборки заказов.

Выполнение

-- Создана бд, добавлены таблицы, а так же добавления хранимой процедуры для расчета времени сборки заказа на основе суммарного времени сборки номенклатуры на сборочной площадке.
-- Перед созданием клиентской части на Visual Studio C# для системы сборки заказа:

1. Создать проект в Visual Studio:

   - Открыть Visual Studio и создать новый проект C# Windows Forms Application.
   - Указать имя проекта и место сохранения.

2. Добавление подключения к базе данных:

   - Перейти в меню "View" -> "Server Explorer".
   - Нажмите правой кнопкой мыши на "Data Connections" и выберите "Add Connection".
   - Выберите ваш сервер баз данных MS SQL, укажите имя сервера, базу данных и проверьте подключение.
   - Нажмите "OK" для добавления подключения.

3. Добавление библиотеки для работы с базой данных:
   - В Solution Explorer выберите ваш проект и нажмите правой кнопкой мыши, затем выберите "Manage NuGet Packages".
   - Установите библиотеку "Entity Framework" для работы с базой данных MS SQL.
